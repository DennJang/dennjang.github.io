网站运维日志

发表于 2022-06-14 分类于 然哥秘籍


本文记录本网站细节的编辑及更新、问题的出现及解决以及一些杂七杂八的感受。

2022年

6月13日 初步框架搭建

1. 环境准备

● 安装 git (版本2.36.1)，也就是传说中世界上最先进最高端的分布式版本控制系统。

官网：https://git-scm.com/

● 安装 node.js，它是一种 javascript 的运行环境，能够使得 javascript 脱离浏览器运行，直接运行在服务端，我的理解是这样会有利于进行本地开发的效率，更何况除了这个我就不会搞别的了。

官网：https://nodejs.org/en/download/
原理：http://www.sitepoint.com/node-js-is-the-new-black/

● 安装 Hexo，它是一个简介快速又高效的博客框架，还有很多强大的插件，对比多个之后我倾向于用这个。

官网：https://hexo.io/zh-cn/index.html

● 安装 Hexo 依赖，即 hexo-deployer-git 插件。这个东西似乎是一个用来一键部署的插件，详细原理我看不明白，先放着呗。

命令：npm install —save hexo-deployer-git

2. 绑定 GitHub

● 绑定 git 到 GitHub 便于在本地上传文件，虽然也可以在 GitHub 上传，但是由于神秘的魔法还有学校不稳定的网络，这并不是一件容易的事情。可以利用 SSH 来完成绑定。

SSH：安全外壳协议(Secure Shell) ，可以比较安全地保护上传的数据，防止信息泄露的问题，而且可以压缩数据，提高传输速度。
原理：https://www.jianshu.com/p/33461b619d53

● 在 Git Bash 中检测是否已经安装了 SSH，若已安装，则指定 RSA 算法生成密钥，生成两个文件，分别叫做 id_rsa 和 id_rsa.pub。

● 把 id_rsa.pub 的内容添加到 GitHub，相当于门锁配对，之后用以下命令检验是否配对成功。注意空格。

命令：ssh -T git@github.com

● 提交文件。将 GitHub 的远程仓库 clone 到本地，添加文件并 commit 到本地仓库，然后再将本地仓库的内容 push 到远程仓库，详细指令在知乎、CSDN、简书都有，我混乱地没按步骤来走，最后却也成功了，可能是为接下来遇到的问题埋下祸根。

3. 初始化博客

● 使用 Hexo 自带的命令进行静态部署，之后可以在浏览器输入终端显示的本地地址查看初始化后的网站样子。至此，博客搭建完毕。

4. 美化博客

● 下载安装了 next 主题，加入标签 (tags) 、分类 (categories) 及关于 (about) 页面，虽然可以创建，但不能直接打开，试了很多方法无法解决，祸根浮现。

6月14日 功能调整

1. 添加上传图片功能

● 添加 hexo-asset-image 插件，按使用说明，它会在生成新文章的同时生成一个同名的空文件夹，在里面放入所需上传的图片，然后编辑 md 文件使用默认指令调用。我试了很多次，都不能成功显示图片，发现本地文件并不能成功上传到 GitHub 的仓库中去，猜测可能需要使用魔法，但也失败，最终看到了下面这篇文章才解决了所有的问题。

好攻略：https://www.jianshu.com/p/f72aaad7b852

2. 添加书签

● 用了自带的 bookmark ，可以根据阅读历史记录，在下次打开页面的时候定位到上次的位置。

3. 添加文章置顶功能

● 安装 hexo-generator-index-pin-top 插件之后，在 /themes/next/layout/_macro 目录下的 post.swig 设置置顶标志，之后在文章添加 top 就行，方法是在 Front-matter 中添加 top: true 。

4. 调整菜单分布

● 鉴于标签 (tags) 、分类 (categories) 及关于 (about) 页面均显示不出的问题，我突发奇想可以把他们放在同一个页面中，即我只需要在一个页面中加入超链接引入子项就行。

● 我的做法是新建一篇置顶文章，作为首页的引导，之后在该文章中可以详细介绍这个网站的细节，包括各个分类是做什么的，当然包括我的一些想法在里面，后续排版美化就慢慢编辑吧。

5. 添加“然哥秘籍”分类

● 这个分类本来是想要作为记录这个网站的运营维护细节的分类，广义化就是我的一些探索后的收获或者所谓的学习心得，后来经过伟大的叶大哥提醒，其实可以作为一个线上学习资料库，因为期末考要到了，把我的一些笔记上传上来也可以方便随时查看阅读。

6月15日 功能调整

1. 更新文章

● 编辑更新“高层建筑结构设计”，属于“然哥秘籍”分类。

2. 部署失败原因

● 定睛一看才发现每次部署失败是因为 “unalble to access github.com/*“，也就是说防火墙难辞其咎，所以连接 VPN 进行部署实属迫不得已，感谢学校伟大的 WiFi 。

3. 修改字体

● 修改电脑端字体为“微软正黑（Minecrosoft Jhenghei）”。

4. 加入公式

● 添加MathJax数学公式，并将 Hexo 渲染引擎“hexo-renderer-marked”改为“hexo-renderer-kramed”，但遇到 bug，暂不明原因，所以又把引擎换回来了。

5. 启用评论

● 添加评论功能，基于 LeanCloud 的 Valine 评论系统。

6月16日 功能调整

1. 修复公式问题

● 修复数学公式显示的问题。出现问题的原因是 MathJax 的官方 cdn 停止维护，最终冲浪找到一个可用的 cdn 地址。此时再将渲染引擎更改为 hexo-renderer-kramed，但这次没有把原来的卸载，却也没有出现问题。此外，也跟着教程改了两个转义规则，解决语法冲突的问题。最后，因为不知道主题是否支持 MathJax，再添加了 hexo-filter-mathjax 工具以防万一。至此，网页成功显示了公式。

2. 编辑文章

● 编辑更新《高层建筑结构设计》，属于“然哥秘籍”分类。

3. 添加脚注功能

● 其实也就是安装了 hexo-reference ，方便以后我写潮汕话文章。
